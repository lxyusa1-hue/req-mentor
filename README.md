# req-mentor

需求智能辅导技能 —— 让 AI 通过对话引导运营人员从模糊想法产出结构化、可执行的需求文档。

支持**系统组**和**开发组**两套需求模板，群聊和私聊均可触发。

## 特性

- **智能识别**：自动识别需求提交意图，非需求消息不触发
- **双模板路由**：先问提交给系统组还是开发组，再按对应模板引导
- **推测式确认**：能推断的直接让用户确认，不逐个重新问，效率高
- **多选优先**：尽量给选项，减少运营人员组织语言的负担
- **完整性检查**：自动检测缺失项，针对性追问补充
- **四维评估**：生成文档后自动评估完整性、清晰度、可行性、优先级合理性
- **飞书集成**：确认后自动创建飞书云文档，放到指定文件夹

## 核心流程

```
识别需求意图 → 确认提交组别 → 引导式对话采集 → 完整性检查 → 生成文档 → 初步评估
```

## 安装

将 `req-mentor` 目录放到 OpenClaw 的 skills 目录下即可：

```bash
# 方式一：克隆
cd ~/.openclaw/workspace/skills/
git clone https://github.com/lxyusa1-hue/req-mentor.git

# 方式二：下载 zip 解压
# 下载后解压到 ~/.openclaw/workspace/skills/ 目录
```

## 目录结构

```
req-mentor/
├── SKILL.md                           # 核心流程与对话规则
└── references/
    ├── system-group.md                # 系统组引导字段（6步）
    ├── development-group.md           # 开发组引导字段（10步 + 版本管理）
    └── output-template.md             # 两组输出格式 + 四维评估模板
```

## 模板覆盖

### 系统组（6个板块）
需求标题 → 背景与问题 → 需求目标 → 业务场景描述 → 功能范围 → 优先级与紧急程度

### 开发组（版本化结构）
需求背景 → 需求类型 → 需求概述（功能模块/逻辑说明/展示方式/数据路径/执行频率/特殊规则）→ 版本迭代管理

## 触发方式

在群聊或私聊中发送包含以下意图的消息：

- 关键词："提个需求""帮我写个需求""我要提个功能"
- 功能期望："我希望系统能…""能不能加个…""想要一个…"
- 问题痛点：描述了一个隐含需要开发支持的问题

## 设计原则

1. **推测式确认** — 能推断的直接确认，让用户点头
2. **多选优先** — 减少用户组织语言的负担
3. **一次一个问题** — 不批量提问，避免信息过载
4. **主动推进** — 根据已有信息跳过/合并问题
5. **禁止岔开** — 温和拉回偏离的需求讨论
6. **停止条件** — 用户说"够了"就直接生成
