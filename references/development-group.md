# 开发组需求模板 — 引导字段

模板参考：[📋 开发组需求文档模板](https://xisande-tp.feishu.cn/docx/YSKrd6Ys9oQSwbxZY5gcLKUJnEf)

## 引导顺序（推测式确认优先）

### 1. 需求标题
- 用户说完后直接整理确认：
> "我理解你要做的是：「______」，是否准确？"

### 2. 需求类型（多选）
```
这个需求属于哪种类型？
1. 全新需求
2. 程序优化（已有功能改进）
3. 数据优化（数据展示/报表类）
```

### 3. 优先级（早期确认）
```
紧急程度？
P1 紧急 / P2 高 / P3 中

期望上线时间：____
```

### 4. 需求目的
- 追问：你希望这个功能最终帮你解决什么问题？
- 有业务价值线索时直接确认：
> "就是为了解决XX对吗？"

### 5. 功能模块（多选）
```
归属哪个模块？
A. 工单数据 B. 工单图表 C. 大甲餐饮 D. 工单工具 E. 自动化程序
```

### 6. 需求内容（核心，推测式确认）

**逻辑说明：**
- 详细拆解每一步，自动化功能写清触发条件、执行逻辑、输出结果
- 用户有描述时合并确认：
> "我总结一下逻辑：触发条件是XX，执行XX操作，结果推送到XX，对吗？"

**展示方式：**
```
数据展示在哪里？
A. 页面/看板  B. 群聊消息  C. 定时邮件  D. 其他（____）
```

### 7. 数据路径
- 数据从哪里来？
- 示例追问："数据是从哪个系统取的？能不能截个图给我看看？"

### 8. 执行频率
```
多久执行一次？
A. 实时  B. 每天  C. 每周  D. 每月  E. 按需触发  F. 其他（____）
```

### 9. 特殊规则
```
有没有特殊规则？
□ 白名单/权限限制（____）
□ 数据筛选条件（____）
□ 特殊处理逻辑（____）
□ 没有特殊规则
```

### 10. 版本管理（迭代需求时触发）

用户提到"改一下""增加""优化"等迭代意图时：
1. 识别为版本迭代，自动新增版本章节
2. 记录：版本编号、日期、编辑人、变更内容
3. 只追问变更部分，不重复已有信息
