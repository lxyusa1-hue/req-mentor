# 文档输出模板

## 系统组输出格式

严格按照[📋 需求文档模板（标准版）](https://xisande-tp.feishu.cn/docx/DQlcdD1ONoFy6xxCimBc71gEn1d)的7个板块：

```markdown
# 需求文档

## 1️⃣ 需求标题
[一句话标题]

## 2️⃣ 背景与问题
**现状：** ...
**痛点：** ...
**业务价值：** ...

## 3️⃣ 需求目标
**解决什么问题：** ...
**期望效果：** ...
**衡量指标：** ...

## 4️⃣ 业务场景描述
| 角色 | 场景 | 输入 | 期望输出 |
|------|------|------|----------|
| ... | ... | ... | ... |

## 5️⃣ 功能范围
**✅ 要做的事：** ...
**❌ 暂不做的事：** ...

## 6️⃣ 优先级与紧急程度
**优先级：** Px
**期望上线时间：** YYYY-MM-DD
**是否有关联需求：** ...

## 7️⃣ 附件
- ...
```

## 开发组输出格式

严格按照[📋 开发组需求文档模板](https://xisande-tp.feishu.cn/docx/YSKrd6Ys9oQSwbxZY5gcLKUJnEf)的结构：

```markdown
### 提交部门：XXX业务部
### 需求优先级：Px（紧急/高/中）
### 期望上线时间：YYYY-MM-DD

| 版本 | 日期 | 编辑人 | 变更内容说明 |
|------|------|--------|-------------|
| V1.0 | ... | ... | 首次提交 |

# 一、需求背景
[背景]

# 二、需求类型
[全新需求/程序优化/数据优化]

# 三、需求概述
## V1.0版本 小标题（需求提交时间）
### 需求目的：...
### 需求内容：...
#### 功能模块：...
#### 逻辑说明：...
#### 展示方式：...
### 数据路径「来源」：...
### 需求执行次数/频率：...
### 特殊规则：...
```

```markdown
- ...

### 建议
- ...
```
