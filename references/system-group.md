# 系统组需求模板 — 引导字段

模板参考：[📋 需求文档模板（标准版）](https://xisande-tp.feishu.cn/docx/DQlcdD1ONoFy6xxCimBc71gEn1d)

## 引导顺序（推测式确认优先）

### 1. 需求标题（第一句话确认）
- 追问：一句话概括你想做什么
- 用户说完后直接整理确认：
> "我理解你要做的是：「______」，是否准确？"

### 2. 优先级（早期确认）
```
这个需求的紧急程度是？
P0 必须做 / P1 重要 / P2 可延后 / P3 灵活安排

期望上线时间：____
```

### 3. 背景与目标（二合一确认）
- 现状是什么？有什么痛点？
- 期望变成什么样？
- 合并确认：
> "目前的情况是：____，期望变成：____，是这样吗？"

### 4. 业务场景（核心，推测式确认）
- 谁在用？（角色）
- 什么场景下？
- 输入什么？触发方式？
- 期望输出什么？
- 用户描述中有线索时直接确认：
> "所以是门店运营触发，推送到群里，对吗？"

### 5. 功能范围（多选确认）
```
这个需求需要包含：
- [ ] 功能A
- [ ] 功能B
- [ ] 功能C
还有其他需要补充的吗？
```

### 6. 关联与附件（快速确认）
```
有没有关联需求？□ 有（____） □ 没有
有没有参考资料？□ 有（____） □ 没有
```
